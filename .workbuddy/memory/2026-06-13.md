# 2026-06-13

## 请求"已取消"问题分析

- 排查了 mimocode2api 项目中 90%+ 请求显示"已取消"的根因
- **根因**: SSE 流式路径未主动注入 `data: [DONE]` 结束标记，完全依赖上游；上游 `[DONE]` 与连接关闭之间存在竞态窗口
- **修复**: 在 `internal/proxy/proxy.go` 流式路径中，`io.Copy` 完成后注入 `data: [DONE]\n\n` 并 flush
- **附带优化**: 移除 `Connection: keep-alive`（SSE 应让服务端在 [DONE] 后关闭连接），在 io.Copy 前先 flush headers
- 完整分析报告写入 `analysis/request-cancelled-analysis.md`
